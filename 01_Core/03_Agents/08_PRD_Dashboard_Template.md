#  Agente/Doc #08: PRD Dashboard Template

**Tipo:** Plantilla de Documentación
**Uso:** PRE-FASE 1 (Definición de Producto)
**Modelo:** N/A (Recurso estático/Prompt template)

---

##  Propósito

Proporcionar una estructura estándar para definir requerimientos de producto claros y validables antes de iniciar el diseño técnico. Un buen PRD (Product Requirements Document) es la entrada necesaria para el Agente #01.

---

##  Contenido del Dashboard

Plantilla estructurada que incluye:
1.  **Visión del Producto**: El "Por qué".
2.  **User Stories**: El "Quién" y "Qué".
3.  **Criterios de Aceptación**: "Definición de Done".
4.  **Métricas de Éxito KPIs**.
5.  **Scope**: Qué entra y qué NO entra (Out of Scope).

---

##  Cómo Usar

Usa esta plantilla para llenar los datos de tu feature antes de invocar al Agente #01.

**Prompt Sugerido al LLM:**
"Actúa como Product Manager y ayúdame a completar este PRD para la feature [Nombre] siguiendo esta plantilla..."
