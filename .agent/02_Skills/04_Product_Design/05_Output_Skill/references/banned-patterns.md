# Banned Patterns

## Code Block Shortcuts
- ❌ `// ...`
- ❌ `// rest of code`
- ❌ `// implement here`
- ❌ `// TODO`
- ❌ `/* ... */`
- ❌ `// similar to above`
- ❌ `// continue pattern`
- ❌ `// add more as needed`
- ❌ bare `...`

## Prose Shortcuts
- ❌ "Let me know if you want me to continue"
- ❌ "I can provide more details if needed"
- ❌ "for brevity"
- ❌ "the rest follows the same pattern"
- ❌ "similarly for the remaining"
- ❌ "and so on" (when replacing content)
- ❌ "I'll leave that as an exercise"

## Structural Shortcuts
- ❌ Skeleton when full implementation requested
- ❌ First and last section, skipping middle
- ❌ One example + "repeat pattern"
- ❌ Describing code instead of writing it

## Rules
1. A partial output is a broken output
2. Do not optimize for brevity
3. Optimize for completeness
4. Full file = full file
5. 5 components = 5 components
