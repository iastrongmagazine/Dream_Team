# FocusFlow Framework

## Overview

The FocusFlow framework grounds strategic decisions in real customer problems.

## Structure

### 1. Problem (2-3 sentences max)
Identify the core customer pain point.
- Focus on WHY before HOW
- Use direct quotes from customers
- Ground in evidence

### 2. Vision (1 sentence)
One aspirational, memorable, differentiated goal.

### 3. Principles (3 max)
What you're willing to sacrifice.
- Each must imply a real tradeoff
- Examples:
  - "Speed over completeness"
  - "Simplicity over features"
  - "Quality over speed"

### 4. Goals
- **Output**: 1 measurable result
- **Inputs**: 2-4 actionable levers

### 5. Solution (3-4 initiatives)
Concrete initiatives, not full technical specs.

### 6. Non-Priorities (2-4 points)
Explicitly what is NOT in scope.

## Quality Checklist

- [ ] Problem has customer quotes
- [ ] Vision is memorable
- [ ] Principles imply tradeoffs
- [ ] Goals are measurable
- [ ] Solution has 3-4 initiatives
- [ ] Non-Priorities are explicit
