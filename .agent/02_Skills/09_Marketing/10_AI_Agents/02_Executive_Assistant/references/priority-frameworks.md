# Priority Frameworks

## Eisenhower Matrix

| | Urgent | Not Urgent |
|---|--------|------------|
| **Important** | Do First (P0) | Schedule (P2) |
| **Not Important** | Delegate (P3) | Eliminate |

**When to use**: General prioritization, especially when overwhelmed.

## ABCDE Method

- **A**: Must do — serious consequences if not done
- **B**: Should do — mild consequences
- **C**: Nice to do — no consequences
- **D**: Delegate — can someone else do this?
- **E**: Eliminate — delete from list

**When to use**: Simple task lists, daily planning.

## ICE Scoring

| Factor | Question |
|--------|----------|
| **I**mpact | How much will this help? (1-10) |
| **C**onfidence | How sure are we it will work? (1-10) |
| **E**ase | How easy to implement? (1-10) |

Score = (I + C + E) / 3

**When to use**: Feature prioritization, project selection.

## RICE Scoring

| Factor | Description |
|--------|-------------|
| **R**each | How many people will this affect? |
| **I**mpact | How much will it help? (0.25/0.5/1/2/3) |
| **C**onfidence | How sure are we? (50%/80%/100%) |
| **E**ffort | Person-weeks to implement |

Score = (R × I × C) / E

**When to use**: Product roadmap prioritization.

## MoSCoW Method

- **Must have**: Non-negotiable
- **Should have**: Important, not critical
- **Could have**: Nice to have
- **Won't have** (this time): Explicitly deprioritized

**When to use**: Release planning, scope definition.

## WSJF (Weighted Shortest Job First)

```
WSJF = (User Business Value + Time Criticality + Job Size Reduction) / Job Size
```

**When to use**: Agile sprint planning, SAFe prioritization.

## Applying Frameworks

| Situation | Recommended Framework |
|-----------|----------------------|
| Overwhelmed, need clarity | Eisenhower |
| Simple task list | ABCDE |
| Feature prioritization | ICE or RICE |
| Release planning | MoSCoW or WSJF |
| Sprint planning | WSJF |
