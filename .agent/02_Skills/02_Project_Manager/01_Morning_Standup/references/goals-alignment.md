# Goals Alignment Guide

## Overview

This guide explains how to align daily tasks with quarterly goals during morning standups.

## The Alignment Matrix

| Goal Priority | Daily Focus | Weekly Minimum |
|---------------|-------------|----------------|
| P0 (Critical) | 60% of day | 4 hours |
| P1 (Important) | 30% of day | 2 hours |
| P2 (Nice to have) | 10% of day | 30 min |

## How to Check Alignment

### Step 1: Map Tasks to Goals

Each task should link to a specific goal:

```
Tarea: "Implementar login con Google"
→ Goal: "User Authentication" (GOALS.md)
→ Quarter: Q1 2026
→ Progress: 40% → 60%
```

### Step 2: Verify Progress Contribution

Ask: "Does completing this task move a goal forward?"

- **YES** → Include in standup
- **NO** → Defer to later in day/week
- **NOT SURE** → Add to backlog for clarification

### Step 3: Calculate Daily Progress

Track how each task contributes to goal percentage:

| Goal | Current % | Today +% | End of Day % |
|------|-----------|----------|--------------|
| Auth System | 40% | +20% | 60% |
| API v2 | 15% | +5% | 20% |

## Common Mistakes

1. **Task without goal link** → Creates busy work, not progress
2. **Too many P2 tasks** → Misses Q1 deadlines
3. **No metric for success** → Can't measure progress
4. **Goals too vague** → Impossible to align tasks

## Quick Reference

```
Alignment Check:
1. Does this task have a goal tag?
2. Does it advance the goal by >5%?
3. Is it P0 or P1 priority?
4. Can it be done in <2 hours?

If all YES → Include in standup
```
