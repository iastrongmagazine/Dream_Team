## Design Sprint

### What It Solves
Validate ideas in 5 days instead of weeks/months. Reduces risk by testing with real users before building.

### When to Use
- New feature validation
- Problem solving
- Quick alignment with stakeholders
- High-risk decisions

### How It Works
| Day | Phase | Focus |
|-----|-------|-------|
| Monday | Map | Define problem, map journey |
| Tuesday | Sketch | Ideate solutions (Crazy 8s) |
| Wednesday | Decide | Choose best solution |
| Thursday | Prototype | Build realistic mockup |
| Friday | Test | Validate with 5 users |

### Example
```
Day 1: "How might we reduce checkout abandonment?"
Day 2: Sketch 3 solutions
Day 3: Pick "One-click express checkout"
Day 4: Build Figma prototype
Day 5: Test with 5 users → 3/5 completed purchase
```

### When NOT Use
- Already validated problem/solution
- Technical implementation only
- Ongoing optimization
