## Lean UX

### What It Solves
Build → Measure → Learn loop with minimum viable products. Focus on outcomes over outputs.

### When to Use
- Startups and MVPs
- Rapid experimentation
- Data-driven decisions
- Limited resources

### How It Works

```
    ┌─────────┐
    │  IDEA   │
    └────┬────┘
         │
    ┌──▼──────────┐
    │   BUILD     │ ← MVP, not full product
    │  (hypothesis)│
    └────┬─────────┘
         │
    ┌──▼──────────┐
    │   MEASURE   │ ← Metrics, not just vanity
    │   (data)    │
    └────┬─────────┘
         │
    ┌──▼──────────┐
    │   LEARN     │ ← Validated learning
    │ (insights)  │
    └────┬─────────┘
         │
    ┌──▼──────────┐
    │  PIVOT or  │ ← Based on evidence
    │  PERSEVERE │
    └───────────┘
```

### Core Principles

1. **Assumptions > Requirements** - List assumptions, prioritize risk
2. **Outcomes > Outputs** - Focus on business metrics
3. **速度 > Perfection** - Ship fast, iterate
4. **Collaboration > Siloes** - Cross-functional teams
5. **Learning > Validation** - Evidence over opinions

### Example

| Hypothesis | Experiment | Metric | Result |
|-----------|------------|--------|--------|
| "Users will add to cart" | A/B test CTA color | Cart add rate | +23% → Keep |

### When NOT Use
- Regulated products (medical, finance)
- Heavy engineering solutions
- When user research is critical
